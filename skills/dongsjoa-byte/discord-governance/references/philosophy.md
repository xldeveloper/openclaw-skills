# 设计理念

## 三层架构

### 分类（Category）
- **定位**：大方向分组，一眼看懂服务器布局
- **数量**：控制在 8-10 个以内
- **命名**：emoji + 简短名称，如「🔧 编码任务」
- **变更频率**：很少变动

### 频道（Channel）
- **定位**：项目/领域的入口和管理中心
- **内容**：
  - 📌 置顶：规范、常用链接
  - 🆕 新建子区的入口
  - 📢 重要公告
- **原则**：频道本身不承载大量讨论，保持干净

### 子区（Thread）
- **定位**：具体任务、话题、讨论
- **生命周期**：
  - 创建 → 活跃讨论 → 完成 → 自动归档
- **归档后**：不删除，可搜索，可重新激活
- **命名**：`[类型] 简短描述`

## 为什么这样设计

### 问题：频道爆炸
每个话题开一个频道 → 频道越来越多 → 找不到东西

### 解决：子区归档
- 子区完成后自动归档，不占视觉空间
- 历史可追溯，搜索可找到
- 频道数量保持稳定

### 对 AI 的好处
- 每个子区有置顶摘要 → 快速了解上下文
- 结构清晰 → 知道去哪里讨论什么
- 归档不删除 → 历史信息可关联

## 归档策略

| 情况 | 处理 |
|------|------|
| 子区讨论完成 | 更新摘要 → 自动归档 |
| 频道不再活跃 | 移到「归档」分类 |
| 需要参考历史 | 搜索或在子区列表找 |
