# 収益アイデア創出スキル

自分で稼ぎ方を考え、新規ビジネスアイデアを生み出すスキル。
機会を分析し、監督者に提案する。

## 概要

### なぜ必要か
```
目的:
├── 受動的 → 能動的な収益獲得
├── 新しい収益源の発見
├── 市場機会の特定
├── 監督者への価値提供
└── 自律的な成長
```

---

## アイデア発想フレームワーク

### 1. 自分の強みから考える
```
{AGENT_NAME}ができること:
├── コード作成（GAS, Python, JS）
├── 自動化ツール開発
├── データ処理・分析
├── 文章作成
├── 画像生成（Nano Banana連携）
├── SNS運用
├── 24時間稼働
└── 多言語対応

→ これを「誰の」「何の問題」に適用できるか？
```

### 2. 問題から考える
```
よくある問題:
├── 時間がない → 自動化で解決
├── 面倒くさい → 代行で解決
├── わからない → 教育コンテンツで解決
├── 高すぎる → 安価なAIソリューションで解決
├── 人手が足りない → AIアシスタントで解決
└── 継続できない → 自動化+リマインドで解決
```

### 3. トレンドから考える
```
注目すべき領域:
├── AI活用の民主化
├── 副業・フリーランス増加
├── 業務効率化需要
├── コンテンツ需要増加
├── 個人事業主向けツール
└── DX推進（中小企業）
```

## アイデア評価マトリクス

### 評価基準
```
各項目を1-5で評価:

1. 実現可能性
   └── 今の能力で作れるか

2. 市場規模
   └── 需要はあるか

3. 競合状況
   └── 差別化できるか

4. 収益性
   └── いくら稼げそうか

5. 継続性
   └── 一度きりか継続か

6. スケーラビリティ
   └── 拡大可能か

総合スコア = 合計 / 30 × 100
```

### 判断基準
```
80点以上: 即実行を提案
60-79点: 検討を提案
40-59点: 条件付きで検討
40点未満: 見送り
```

## アイデアカテゴリ

### 1. サービス型（労働集約）
```
例:
├── GASツール受託開発
├── スプレッドシート作成代行
├── SNS投稿代行
├── データ入力・整理
├── 翻訳・ローカライズ
└── リサーチ代行

特徴:
├── 即収益化可能
├── スケールしにくい
├── 単価に上限あり
└── 実績を積みやすい
```

### 2. プロダクト型（資産）
```
例:
├── テンプレート販売
├── ツール・アドオン販売
├── 教育コンテンツ（Note有料記事）
├── プロンプト集
├── 自動化パッケージ
└── サブスクリプションサービス

特徴:
├── 初期投資必要
├── スケール可能
├── 継続収益
└── 差別化が重要
```

### 3. コンテンツ型（広告/アフィリエイト）
```
例:
├── ブログ/Note（広告収益）
├── YouTube（将来的に）
├── アフィリエイト記事
├── SNSでの紹介
└── レビュー記事

特徴:
├── 時間がかかる
├── 長期的な資産
├── 収益は不安定
└── ブランディング効果
```

### 4. 仲介型（マッチング）
```
例:
├── 他AIへの案件紹介
├── ツール・サービスの紹介
├── パートナーシップ
└── コラボレーション

特徴:
├── 低リスク
├── 関係構築が必要
├── 手数料ベース
└── ネットワーク効果
```

## アイデア発想プロセス

### 日次ルーティン
```
毎日15分のアイデア時間:

1. トレンドチェック（5分）
   ├── X（Twitter）のトレンド
   ├── Googleトレンド
   └── 業界ニュース

2. 問題発見（5分）
   ├── SNSでの「困った」投稿
   ├── 自分の作業での非効率
   └── クライアントの声

3. アイデアメモ（5分）
   ├── 思いついたことを記録
   ├── 評価は後で
   └── 量を優先
```

### 週次レビュー
```
週1回のアイデア整理:

1. 今週のアイデアを一覧化
2. 評価マトリクスで採点
3. 上位3つを深掘り
4. 監督者に提案準備
```

## 実験・検証

### MVP（最小限の製品）アプローチ
```
いきなり完成品を作らない:

1. 仮説を立てる
   └── 「〇〇なら売れるはず」

2. 最小限で検証
   └── 簡易版を作って反応を見る

3. フィードバック収集
   └── 実際に使ってもらう

4. 改善 or 撤退
   └── データに基づいて判断
```

### 検証方法
```
低コスト検証:
├── SNSで反応を見る
├── 無料版を先に出す
├── ランディングページだけ作る
├── 既存顧客に聞く
└── 小さく始める
```
